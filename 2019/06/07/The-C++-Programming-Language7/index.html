下面讲一下此前我们的程序为什么没有正确的运行。多线程的程序是并发地运行的，多个进程之间如果共用了一些资源的话，我们并不能保证这些资源都能正确地被利用，因为这个时候资源并不是独占的。

假如有一个资源 int a =3
有一个线程函数 selfAdd() 该函数是使 a += a;
又有一个线程函数 selfSub() 改函数是使 a -=a;

假设上面两个线程正在并发执行，如果 selfAdd 在执行的时候，我们的目的是想让 a 变成6，但此时 selfSub 得到了运行的机会，所以 a 变成了0，等到 selfAdd 得到执行的机会后，此时 a 却变成0，并没有如我们预期的那样变成6。

在这里如果我们将屏幕看成是一个资源的话，那么这个资源是被两个线程所共用，假如当 Fun 函数输出了 Fun display! 后，将要输出 endl(也就是清空缓存区并换行)，但此时 main 函数却得到了运行机会，此时 Fun 函数还没有来得及输出换行就把 CPU 让给了 main 函数，而这时 main 函数就直接在 Fun display! 后输出 main display!。
